# Cortex

**Generate a `.claude/docs/` knowledge base for your project — from git history, static analysis and security audit.**

When Claude Code opens a file it immediately knows the full context: historical bugs, pitfalls, related files, tests to run, and security issues. No manual documentation needed.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/kovalenko-tech/cortex/main/install.sh | bash
```

## Quick Start

```bash
cd your-project
cortex analyze
```

Commit the generated `.claude/docs/` to your repo. Done.

## What Claude Code sees

Before writing a single line of code, Claude Code runs:

```bash
python .claude/get_context.py src/auth.py
```

And gets:

```
# src/auth.py

## Historical Insights
- [Bug Fix] 2024-03-15: Session fails on UTC+0 timezone
  Fix: Always use datetime.timezone.utc

## Edit Checklist
- Run: pytest tests/test_auth.py -v
- Check constant: SESSION_TIMEOUT in config/settings.py

## Pitfalls
- Never call authenticate() before db.init()
  Why: connection pool not initialized → RuntimeError

## Related Files
- tests/test_auth.py [co-change: 92%]
- models/user.py [co-change: 67%]

## Security Notes
- ✅ No issues found
```

None of this is written by hand — Cortex extracts it from your git history and codebase.

## CLI

```bash
cortex analyze                  # full analysis
cortex analyze --since HEAD~50  # incremental (last 50 commits)
cortex context src/auth.py      # context for one file
cortex security                 # security audit only
cortex install-hook             # install git pre-commit hook
cortex watch                    # auto-update when new commits arrive
```

## LLM-Enhanced Summaries

Set `ANTHROPIC_API_KEY` to get AI-generated file overviews in every `.md` doc:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
cortex analyze
```

Each file will include a 2-3 sentence technical summary generated by Claude Haiku — no manual documentation needed.

## Auto-update on commit

```bash
cortex install-hook   # installs git pre-commit hook
```

After this, every `git commit` will automatically update `.claude/docs/` for staged files and include those docs in the commit.

## Watch mode

```bash
cortex watch          # auto-updates when new commits arrive
cortex watch --interval 60  # poll every 60 seconds
```

## GitHub Action

Add `.github/workflows/cortex.yml` to auto-update `.claude/docs/` on every push to main/master/develop.
See [cortex.yml example](.github/workflows/cortex.yml).

The action commits updated context back to your repo automatically — so Claude Code always has fresh context without any manual steps.
